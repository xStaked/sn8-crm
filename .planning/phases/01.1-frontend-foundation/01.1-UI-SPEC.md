## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between icon and label |
| sm | 8px | Compact element spacing, conversation list item internal padding |
| md | 16px | Default element spacing, card padding, form field gaps |
| lg | 24px | Section padding, sidebar nav group gaps |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Major section breaks (e.g. login form vertical padding) |
| 3xl | 64px | Page-level vertical centering offset |

Exceptions: Touch targets for sidebar nav icon buttons minimum 44px height (accessibility — default)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 400 (regular) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.2 |

Notes:
- Body (14px/400/1.5): conversation preview text, form input text, panel body text
- Label (12px/400/1.4): sidebar nav item text, metadata (timestamps, contact tags), input labels — visual distinction from Body achieved via smaller size (12px vs 14px) and tighter line height (1.4 vs 1.5)
- Heading (20px/600/1.2): panel section titles, page titles within the shell (e.g. "Inbox")
- Display (28px/600/1.2): login page title "CRM" — only used on login/auth screens

Font stack: `font-sans` (Geist Sans → system-ui → sans-serif). Source: default for Next.js App Router projects.

## Color

Dark theme — all values reference shadcn dark mode CSS variables (HSL tokens, not hardcoded hex). The CSS variable approach from `dashboard-01` is the contract; these HSL equivalents are reference only.

| Role | shadcn Token | Approx Hex | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `--background` | #09090b (zinc-950) | Page background, conversation list background |
| Secondary (30%) | `--card` / `--sidebar` | #18181b (zinc-900) | Sidebar nav, conversation panel, card surfaces, login card |
| Accent (10%) | `--primary` | #fafafa (zinc-50) | Active nav item highlight, selected conversation row, primary CTA button |
| Destructive | `--destructive` | #ef4444 (red-500) | Logout confirmation only |
| Muted text | `--muted-foreground` | #71717a (zinc-500) | Conversation preview truncated text, timestamps, placeholder text |
| Border | `--border` | #27272a (zinc-800) | Column dividers, card borders, input borders |

Accent reserved for: active sidebar navigation item, currently selected conversation row background, primary login button, focus ring on interactive elements.

Source: CONTEXT.md — "Dark theme obligatorio — paleta negra/oscura, minimalista", "Sin colores de acento vistosos — UI discreta, funcional"

## Layout Contract

Source: CONTEXT.md — 3-column layout decision locked.

### Shell Layout (authenticated routes)

```
┌─────────────────────────────────────────────────────────┐
│  Sidebar (56px wide)  │  Conversation List (320px)  │  Detail Panel (flex-1)  │
│  icon nav only        │  scrollable list            │  selected conversation  │
└─────────────────────────────────────────────────────────┘
```

- Sidebar: 56px fixed width, icon-only navigation (no text labels), icons from lucide-react
- Conversation list: 320px fixed width, scrollable, sticky header with page title
- Detail panel: `flex-1`, fills remaining viewport width

Primary visual anchor: the conversation list (320px column) — it carries the most information density and drives all user actions in the inbox view.

### Responsive behavior

Web-first. On viewport < 768px: conversation list and detail panel stack vertically (list visible by default, detail panel slides in when a conversation is selected). Not a primary concern for Phase 1.1 — must not be broken, but does not need to be pixel-perfect.

### Login page

Centered card on full-screen background. Card width: 360px max. Vertically centered with `min-h-screen flex items-center justify-center`.

## Interaction States

All interactive elements must declare these 4 states:

| Element | Default | Hover | Active/Selected | Disabled |
|---------|---------|-------|-----------------|----------|
| Sidebar nav icon | `text-muted-foreground` | `text-foreground bg-accent/10` | `text-foreground bg-accent/20` | not applicable |
| Conversation row | `bg-transparent` | `bg-zinc-800/50` | `bg-zinc-800` with left accent border 2px | not applicable |
| Login button | `bg-primary text-primary-foreground` | `bg-primary/90` | `bg-primary/80` | `opacity-50 cursor-not-allowed` |
| Form input | `border-border` | `border-border` | `ring-2 ring-ring` (focus) | `opacity-50` |

## Route Contract

| Route | Protection | Layout | Primary Action |
|-------|-----------|--------|---------------|
| `/login` | Public — redirect to `/dashboard` if authenticated | Auth (centered card, no shell) | Submit credentials |
| `/dashboard` | Protected — redirect to `/login` if unauthenticated | Shell (3-col) | Default to inbox view |
| `/dashboard/inbox` | Protected | Shell (3-col) | Browse conversations |

Middleware: Next.js `middleware.ts` at `apps/web/middleware.ts`, matches `/dashboard/:path*`, calls `GET /auth/me` to verify session.

Source: CONTEXT.md — "Post-login exitoso: siempre a `/dashboard`", "protección de rutas con middleware".
