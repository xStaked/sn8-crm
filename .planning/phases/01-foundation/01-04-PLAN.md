       - resolves idempotency key from `X-Kapso-Idempotency-Key` then `payload.message.id`

       - emits a structured `webhook_duplicate_skipped` log event when a duplicate delivery is detected

       - missing idempotency key does not enqueue
       - enqueue failure deletes Redis key and raises retryable error
