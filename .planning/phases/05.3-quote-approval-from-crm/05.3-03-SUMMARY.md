# Phase 05.3 Plan 03: CRM quote approval shell integration Summary

**Pending quote signals now surface in the CRM inbox, and the detail panel can preview, approve, or send revision requests for the latest draft without leaving the existing shell.**

## Performance

- **Duration:** 12 min
- **Started:** 2026-04-01T21:00:00Z
- **Completed:** 2026-04-01T21:12:33Z
- **Tasks:** 2
- **Files modified:** 5

## Accomplishments

- Extended the frontend conversation contracts so the shell can carry pending quote summary metadata and full quote review detail with explicit UI states.
- Added a dedicated SWR hook for `/conversations/:conversationId/quote-review` with the same auth-expiry redirect behavior and 5-second refresh cadence used elsewhere in the CRM.
- Integrated pending-review affordances into the inbox and added an approval card above the existing message history/composer stack with `Aprobar y enviar` and `Solicitar revisión` actions wired to the backend.

## Files Created/Modified

- `apps/web/src/types/conversation.ts` - Added pending quote summary, quote review detail, and explicit review state unions for the shell.
- `apps/web/src/hooks/use-conversations.ts` - Preserved the existing `/conversations` fetch flow while widening types and exposing `mutate` for list revalidation after review actions.
- `apps/web/src/hooks/use-conversation-quote-review.ts` - Added SWR-backed quote review fetching with 401 redirect handling and 404-to-empty state mapping.
- `apps/web/src/components/shell/conversation-list.tsx` - Added inline pending-review metadata and badge treatment for conversations that need quote approval attention.
- `apps/web/src/components/shell/detail-panel.tsx` - Added the quote review card, preview area, revision comments flow, and approve/request-changes mutations while preserving the existing manual reply composer below the history.

## Decisions Made

- Returned `mutate` from `useConversations()` so quote approval actions can refresh inbox metadata immediately after a backend state transition.
- Kept the quote review UI inside `DetailPanel` as a sibling section above the message history instead of introducing a route, modal, or shell redesign.
