## Decisions Made

- Kept quote review reads under `/conversations` so the CRM can reuse the selected-conversation model already shipped in Phase 05.
- Treated only `pending_owner_review` and `ready_for_recheck` as inbox-pending states, while still allowing detail reads for `changes_requested`, `approved`, and `delivered_to_customer`.
- Narrowed the conversations e2e module composition to the auth and conversations boundary because Redis/Bull infrastructure is unrelated to this read contract and was blocking deterministic verification.
