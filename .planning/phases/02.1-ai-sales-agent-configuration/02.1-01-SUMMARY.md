# Phase 02.1 Plan 01: AI Sales Domain Foundation Summary

**Conversation-keyed AI sales persistence, a dedicated DeepSeek-backed provider module, and an explicit pending owner quote-template contract**

## Performance

- **Duration:** 3 min
- **Started:** 2026-03-20T01:39:53Z

- **Tasks:** 3

## Accomplishments

- Added durable Prisma models for `CommercialBrief`, versioned `QuoteDraft`, and `QuoteReviewEvent` so brief/draft/review state no longer depends on transcript parsing alone.
- Introduced a dedicated `ai-sales` Nest module, a stable `AiProvider` contract, and a DeepSeek client that owns its own env/config surface.
- Captured the quotation format as a first-class contract file and documented the missing owner-provided template inputs instead of hardcoding an invented format.

## Files Created/Modified

- `prisma/schema.prisma` - Adds conversation-keyed commercial brief, quote draft, and review-event models plus explicit review status enums.
- `apps/backend/src/ai-sales/ai-provider.interface.ts` - Defines the stable provider contract for brief extraction and quote generation/regeneration.
- `apps/backend/src/ai-sales/ai-sales.module.ts` - Wires the AI sales module and provider binding into Nest.
- `apps/backend/src/ai-sales/ai-sales.service.ts` - Exposes orchestration-friendly methods that depend on the provider abstraction.
- `apps/backend/src/ai-sales/deepseek/deepseek.client.ts` - Localizes DeepSeek auth, HTTP calls, JSON parsing, and model selection.
- `apps/backend/src/ai-sales/prompts/sales-agent.system.ts` - Stores the premium consultive system prompt separately from transport code.
- `apps/backend/src/ai-sales/prompts/quote-template.contract.ts` - Declares the owner quotation contract surface and pending inputs.
- `apps/backend/src/app.module.ts` - Imports the dedicated `AiSalesModule`.
- `.env.example` - Documents required DeepSeek runtime variables.

## Decisions Made

- AI sales persistence uses standalone tables keyed by `conversationId` so later plans can read/write reviewable business state without mutating `Message`.
- Review blocking is represented through `QuoteReviewStatus` and `QuoteReviewEvent` records so approval gates can be enforced structurally.
- The missing owner quotation template is left as a typed placeholder contract; `SALES-AI-03` should remain pending until the real template is provided.
