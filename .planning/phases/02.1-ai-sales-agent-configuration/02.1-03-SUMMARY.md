# Phase 02.1 Plan 03: WhatsApp-first owner review, regeneration lineage, and approved-delivery boundary Summary

**WhatsApp-first owner approval commands, queued draft regeneration from socio feedback, and an explicit boundary that keeps customer delivery blocked until approval**

## Accomplishments
- Sent actionable internal quote-review payloads to the socio over WhatsApp before the CRM dashboard approval UI exists.
- Added explicit owner `APPROVE` and `REVISE` commands tied to a conversation and draft version, with durable approval and feedback persistence.
- Regenerated quote drafts through the ai-sales worker queue while preserving prior versions, review events, and a hard no-send boundary for customer delivery.

## Decisions Made
- WhatsApp is the canonical owner-review channel in v1 because it already exists before the CRM dashboard approval surface.
- Owner review commands must be explicit text commands rather than inferred intent so approvals stay durable and attributable to a version.
- Future customer-delivery steps must consume an approved-delivery preparation method instead of reading raw draft rows directly.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Cast regenerated draft payloads to Prisma JSON input types**
- **Found during:** Task 2 (Regenerate drafts from owner feedback while preserving version history)
- **Issue:** Prisma rejected the regenerated `draftPayload` object because nested structured data was not explicitly typed as `InputJsonValue`.
- **Fix:** Built the payload as a `Prisma.InputJsonObject` and cast nested structured fields explicitly.
- **Files modified:** `apps/backend/src/ai-sales/owner-review.service.ts`
- **Verification:** `pnpm build`
- **Committed in:** `c5785b8`

**2. [Rule 3 - Blocking] Normalize loosely typed webhook text/from fields before owner command parsing**
- **Found during:** Task 1 (Notify the socio and expose an explicit review command contract)
- **Issue:** `KapsoWebhookDto` exposes nested message text fields as loose shapes, so direct property access broke the TypeScript build.
- **Fix:** Added typed extraction helpers in `WebhooksService` before handing payloads to the owner-review command parser.
- **Files modified:** `apps/backend/src/webhooks/webhooks.service.ts`
- **Verification:** `pnpm build`
