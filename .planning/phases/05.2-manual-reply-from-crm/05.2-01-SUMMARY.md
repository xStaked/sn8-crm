# Phase 05.2 Plan 01: Backend manual reply send path Summary

**Manual CRM replies now send through Kapso with the real provider `wamid`, persist as outbound Message rows, and return an authenticated conversation-scoped response contract for the frontend.**

## Performance

- **Duration:** 14 min
- **Started:** 2026-03-19T03:40:00Z
- **Completed:** 2026-03-19T03:54:06Z
- **Tasks:** 2
- **Files modified:** 8

## Accomplishments

- Threaded the Kapso `wamid` through `KapsoClient`, `KapsoAdapter`, `ChannelAdapter`, and `MessagingService` so outbound sends expose the real provider id.
- Added `SendMessageDto` plus an authenticated `POST /conversations/:conversationId/messages` route on the existing conversations controller.
- Persisted outbound manual replies into the existing `Message` table and returned the created conversation message payload for immediate frontend rendering.

## Files Created/Modified

- `apps/backend/src/channels/channel.adapter.ts` - Changes the outbound text contract to return the provider message id.
- `apps/backend/src/channels/kapso/kapso.client.ts` - Captures the Kapso SDK response and returns `messages[0].id`.
- `apps/backend/src/channels/kapso/kapso.adapter.ts` - Forwards the returned `wamid` through the channel abstraction.
- `apps/backend/src/messaging/messaging.service.ts` - Exposes the returned outbound message id to callers.
- `apps/backend/src/conversations/dto/send-message.dto.ts` - Defines the validated request body for manual replies.
- `apps/backend/src/conversations/conversations.service.ts` - Sends the outbound message, persists the row, and maps the created record back to the conversation message contract.
- `apps/backend/src/conversations/conversations.controller.ts` - Adds the authenticated POST route and Swagger metadata for manual replies.
- `apps/backend/src/conversations/conversations.module.ts` - Imports `MessagingModule` so the conversation service can send through the existing channel stack.

## Decisions Made

- Used `KAPSO_PHONE_NUMBER_ID` as the preferred outbound sender identity for persisted rows so future conversation logic can distinguish sender vs participant without schema changes.
- Kept the response payload aligned with the existing `ConversationMessageDto` contract so the detail panel can append the sent message without a new frontend type.
