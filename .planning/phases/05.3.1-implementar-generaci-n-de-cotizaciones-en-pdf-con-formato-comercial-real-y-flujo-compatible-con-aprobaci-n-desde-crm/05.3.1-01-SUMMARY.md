## Decisions Made

- Used a per-`QuoteDraft.id` artifact row as the cache key so regenerated versions never reuse an older PDF.
- Kept customer delivery untouched; the new PDF foundation only prepares a read-side artifact for CRM review.
- Loaded `@react-pdf/renderer` dynamically inside the service to avoid breaking the CommonJS NestJS backend with an ESM-only package.

## Issues Encountered

- The reference PDF title metadata was readable in this environment, but the document text was not extractable through local CLI tools. The contract therefore locks fallback wording explicitly instead of inventing it silently at render time.
- `@react-pdf/renderer` is ESM-first. The implementation isolates loading inside `QuotePdfService` so the current CommonJS backend remains compatible.
