# Commercial Quote PDF Contract

## Locked Decisions

- Source of truth: `Cotizacion - Norgtech.pdf` is the reference artifact for tone and commercial intent. In this workspace the file metadata is readable but its text content is not machine-extractable, so every undefined item below is locked here as an explicit fallback instead of being guessed later in code.
- Layout scope for v1: a single commercial quote PDF rendered server-side in the backend with a stable section order, no browser runtime, and one persisted artifact per `QuoteDraft.id`.
- Title wording fallback: the visible document title is locked to `Cotizacion Comercial` and must be paired with the draft version badge `Version {n}`.
- Approval-flow wording fallback: every generated PDF must include the internal-review status label `Documento generado para revision interna de SN8 Labs. No enviar al cliente sin aprobacion.`
- Currency fallback: all monetary values are rendered in `COP` with Colombian thousands separators and no decimal cents unless the source data explicitly includes decimals.
- Footer fallback: because the reference PDF text could not be extracted, the legal/commercial footer is locked to a neutral SN8 disclaimer instead of leaving the renderer to improvise.

## Mandatory Sections

Visible order is fixed and must not drift between versions:

1. Header with SN8 Labs brand, document title `Cotizacion Comercial`, and draft version label.
2. Customer and project identity block.
3. Executive summary / project context block.
4. Commercial proposal body rendered from structured sections when available.
5. Pricing / investment summary block.
6. Internal review status notice.
7. Legal / commercial footer.

The customer/project identity block must include, when present:

- `Cliente`
- `Proyecto`
- `Fecha de generacion`
- `Version`

The proposal body must prefer `structuredDraft.ownerReviewDraft.sections` in stored order. If absent, it falls back to:

1. `draftPayload.summary`
2. `renderedQuote`

## Branding Rules

- Brand owner is `SN8 Labs`.
- Header must present the brand before any customer-specific content.
- Visual tone must be commercial and formal, not conversational chat transcript styling.
- The PDF must avoid invented logos, slogans, certifications, or legal registrations that are not present in repo data.
- If no brand palette is explicitly available from data, use neutral black / dark gray text with restrained accent usage suitable for B2B quoting.

## Currency and Number Formatting

- Currency is locked to `COP`.
- Render totals with `COP` prefix, grouped thousands, and comma-free decimal omission by default. Example: `COP 12.500.000`.
- If the source contains decimals, preserve up to two decimal digits.
- Version labels are integers and must render as `Version {n}`.
- Any missing numeric pricing value must render as `Por definir en revision` rather than `0`.

## Legal / Commercial Footer

Mandatory footer content for v1:

- `Documento para revision interna y aprobacion comercial de SN8 Labs.`
- `Los valores y alcances aqui descritos corresponden a la version indicada de la cotizacion.`
- `La entrega al cliente final solo puede ocurrir a traves del flujo aprobado por OwnerReviewService.`

If future phases replace this footer with customer-approved legal copy, they must update this contract first.

## File Naming

- Persisted filename pattern is locked to `cotizacion-sn8-{conversationId}-v{version}.pdf`.
- The filename must be derived from the `QuoteDraft` being rendered, never from mutable status labels.
- Regenerated drafts create a new filename because they create a new `version`.

## Executor Notes

- This contract is mandatory input for all later implementation tasks in Phase `05.3.1`.
- Rendering code must not invent extra sections, branding claims, or delivery semantics outside this document.
- The PDF is a read-side artifact for CRM review. It does not authorize, trigger, or replace customer delivery.
