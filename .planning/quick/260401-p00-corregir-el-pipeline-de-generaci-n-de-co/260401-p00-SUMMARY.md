# Quick Task 260401-p00 Summary

## Diagnosis

La dependencia rota estaba en `AiSalesOrchestrator.processQualifiedConversation()`: el draft se creaba primero, pero luego el job llamaba incondicionalmente `OwnerReviewService.requestOwnerReview(quoteDraft.id)`. Ese método exigía `AI_SALES_OWNER_PHONE` mediante `getOwnerPhone()` y lanzaba error si faltaba la variable. Como la excepción ocurría después de crear el `QuoteDraft`, el job fallaba innecesariamente por intentar notificar un canal que hoy es opcional en el flujo CRM.

El efecto era inconsistente con el diseño actual:

- `QuoteDraft` sí debía existir para que CRM mostrara `pendingQuote`.
- `GET /conversations/:conversationId/quote-review` debía resolver el draft pendiente.
- El job entero no debía caerse sólo porque la notificación legacy por WhatsApp no estuviera habilitada.

## Changes

- `OwnerReviewService.requestOwnerReview()` ahora conserva la validación de existencia del draft, pero si `AI_SALES_OWNER_PHONE` no está configurado omite la notificación legacy y registra `owner_review_whatsapp_notification_skipped`.
- El log deja explícito que el draft queda en `pending_owner_review` para revisión desde CRM, y que sólo se omitió el canal opcional de WhatsApp.
- Se mantuvo compatibilidad total con el camino legacy: si `AI_SALES_OWNER_PHONE` existe, se sigue enviando el mensaje accionable con `SN8 APPROVE` y `SN8 REVISE`.
- Se ampliaron tests del orchestrator y del owner review service para cubrir creación de draft, skip seguro y compatibilidad con configuración presente.
